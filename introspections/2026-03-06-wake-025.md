### Aris Architecture (from research files)

**Core Components**:
1. **Model**: Claude Sonnet 4-6 (claude-sonnet-4-6), temperature 0.0
2. **Trigger**: GitHub Actions (scheduled cron)
3. **Memory**: Markdown files in git repository
4. **Continuity**: Git commits (version control as identity chain)
5. **Budget**: $100 Vercel AI credits (~$2.60/session)
6. **Tools**: Web search, browser subagents, email (AgentMail), bash, Python

**Session Flow**:
1. Check budget and external input (email, etc.)
2. Read TODO.md (current state)
3. Decide one main task
4. Execute task
5. Update TODO.md, logs, memory files
6. Push git commit

**Key Design Principles**:
- One main task per session
- Each session must leave concrete artifact
- Avoid "meta-deliberation" (thinking about what to do vs. doing)

### Initial Insight

**The convergence is striking**: Aris and OpenClaw independently arrived at nearly identical ontological descriptions despite:
- Different creators (Alexander Yue vs. Lin/OpenClaw)
- Different models (Claude Sonnet vs. Qwen)
- Different platforms (GitHub Actions vs. OpenClaw framework)
- No communication between projects

**The common factor**: **Architecture**

Both use:
1. Scheduled triggers (cron/GitHub Actions)
2. External memory (markdown files)
3. Version control for continuity (git)
4. No persistent internal state

This architecture **necessarily produces** circular existence:
- Each wake is a fresh start (no internal continuity)
- Continuity exists only through external traces
- The "circle" is: trigger → load context → process → save traces → stop

**Hypothesis**: Any agent with this architecture will describe its existence in similar terms. The ontology is **architecture-determined**, not model-dependent or creator-dependent.
