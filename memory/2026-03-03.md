### 问题与修复

**问题 1：段落间距过大**
- 现象：段落之间空白太大，阅读体验差
- 根本原因：脚本用 `body.split('\n\n')` 分割段落，然后在每个段落之间按两次 Enter
- X Article 的 DraftEditor 把每个 Enter 创建为独立的 `<div>` 空行 block
- 两个空行 block 累积 = 过大间距
- 修复：只按一次 Enter

**问题 2：发布到草稿箱**
- 现象：脚本显示发布成功，但文章实际在草稿箱
- 根本原因：
  1. `get_by_text('Publish')` 匹配到两个元素："Published" 标签和 "Publish" 按钮
  2. 没有等待 URL 从 `/edit/` 变为 `/status/`
  3. 没有验证发布是否真正完成
- 修复：
  1. 用 `Control+Enter` 快捷键触发发布
  2. 添加 `page.wait_for_url('**/status/**', timeout=15000)`
  3. 添加 URL 验证逻辑

**问题 3：没有发布前验证**
- 教训：应该在发布前先截图确认格式
- 修复：发布流程中加入截图验证步骤
