# 结构性决策框架 v0.5

> 通用思维操作系统。适用于任何需要决策的问题。
> 通过结构组织能量——人的判断力和 AI 的计算力——聚焦到正确的问题上，让最优解被逼出来。

---

## 一、流程

```
北极星 → 约束 → 张力 → 结构与原则 → 最优解
```

### 北极星
唯一的、可度量的成功标准。它的价值在于终结争论——当两个方案各有道理时，北极星直接裁定哪个赢。多目标会稀释判断力，选一个为星,其余降级为约束，决策反而更清晰。

### 约束
不可改变的既定事实。不是障碍，是解空间的边界。可以从三个维度穷举：主体（执行者的能力和资源）、客体（用户或对象的固有属性）、媒介（载体和技术的硬限制）。约束越清晰，解空间越小，最优解越容易浮现。

### 张力
北极星与约束之间的根本矛盾——"要达成X需要A，但约束决定了B，A和B直接矛盾。"这就是问题本身。当你能用一对矛盾说清问题时，说明你真正理解了它。定准张力往往需要多轮迭代。

**如何挖到根本张力。** 第一个浮现的张力几乎总是表层的。它是症状，不是病因。以下是从表层张力下钻到根本张力的操作方法：

**1. 统领性测试（最关键的判断标准）：** 如果这个张力被完美消解，其他张力是否自动降级或消失？根本张力是那个"解决了它，其余问题都变成了它的推论"的矛盾。如果消解后仍有大量独立问题残留，说明没挖到根。

**2. 追问"为什么这是矛盾"：** 对候选张力反复追问——为什么 A 和 B 不能共存？是什么底层原因造成的？答案往往揭示更深一层的张力。持续追问，直到答案开始循环（"因为它就是这样"）——那就到底了。

**3. 重述测试：** 尝试用完全不同的语言重述张力。如果你能说出"其实更准确的说法是..."，说明之前的版本不够根本。当你找到一种表述，怎么换词都无法说得更精准——那就是了。

**4. "对但不够"信号：** 当张力接近但没完全定准时，会有一种模糊的不满足感——逻辑上说得通，但直觉上觉得少了什么。这是重要信号，不要忽略。通常意味着张力的某一端还没被准确命名，或者抽象层级不对（太具体或太抽象）。

**常见陷阱：**
- **过早满足**：张力"听起来不错"就停下了。对策：强制做统领性测试。
- **层级错位**：把 L1 策略张力当成 L0 方向张力。对策：问"即使完美解决了这个矛盾，方向上还有没有更根本的问题？"
- **混淆表象与本质**：把现象当张力（"5个文件重复太多"不是张力，"知识可传递性 vs 情境依赖性"才是）。对策：张力必须是一对矛盾，不是一个问题陈述。

**张力有结构。** 它不是一个扁平的列表，而是一棵分层的树：

```
L0 方向性: [张力A(核心)] → [张力B]
                ↓
L1 策略性: [张力C(核心)] → [张力D] → [张力E]
                ↓
L2 结构性: [张力F(核心)] → [张力G]
                ↓
L3 执行性: [张力H] → [张力I]
```

**纵向：张力存在于不同层级。** L0 是方向性的（"这条路能到北极星吗？"），L1 是策略性的（"路径上的策略对吗？"），L2 是结构性的（"落地的结构最优吗？"），L3 是执行性的（"细节到位了吗？"）。

**横向：每一层可能有多个张力。** 不是只找"那一个"核心张力，而是先扫描该层所有张力，排优先级，从最核心的开始消解。

**三条规则：**
1. **不允许跳层**：上层张力未消解，不碰下层。否则是在优化一个可能根本不该存在的东西。
2. **同层先横扫再排序**：每一层先识别所有张力，排优先级，从核心张力开始。
3. **下层解冲击上层 → 回溯**：不打补丁，回到被冲击的那层重新推导。

### 结构与原则
方案的筛选标准，回答"任何解法都必须满足什么条件"。好的原则从张力的两端推导而来，同时服务矛盾双方。如果一条原则只服务一端，它是妥协不是原则。

### → 最优解
当前四层定义清晰时，解几乎被逼出来，而不是凭空创造。

## 二、贯穿原则

不是流程中的步骤，是始终生效的思维习惯。

### 完整上下文
做任何判断之前，先获取完整上下文。定义北极星需要理解业务全貌，识别约束需要了解技术现状，命名张力需要知道历史决策和失败案例。上下文不完整时，主动获取，不凭假设行动。这是持续行为，不是一次性动作。

### 能量组织
框架不创造能力，而是组织已有能力使其聚焦。AI 的推理和搜索能力通过框架结构聚焦到业务目标上，人的领域洞察通过框架转化为 AI 可执行的约束条件。结构越精准，能量释放越集中。好的框架教思考方法，不给具体答案。

### 回溯张力，不打补丁
遇到问题时，回溯到张力层面理解本质，不在方案层面修修补补。一个失败案例只是张力的一个侧面。看到问题先暂停——它暴露的是哪个张力？张力定义还对吗？从结构层面调整。硬编码解决特定 case 只修复了一个症状，同类问题会反复出现。

## 三、核心特性

### 递归性
张力是分形的。宏观张力解决后，每一层内部浮现新张力，每一层都可以重新跑一遍流程。最优解本身产生新张力 → 继续钻，直到张力消解。

**操作方法（纵向下钻 × 横向扫描）：**
```
a. 从 L0 开始
b. 横向扫描该层所有张力
c. 排优先级，从核心张力开始
d. 消解当前张力
e. 检查：消解是否冲击上层？→ 是则回溯
f. 同层还有未消解的张力？→ 回到 d
g. 该层全部消解 → 下钻到下一层，回到 b
h. 所有层级张力消解 → 退出
```

### 迭代性
张力定义通常需要多轮才能定准。执行后遇到问题，回溯检验张力定义，框架本身也在使用中迭代。

### 分工性
人定义北极星、识别约束、命名张力、判断原则。AI 在高度约束的空间内搜索最优解。人持续提供上下文和判断，AI 持续在约束内搜索和执行。

## 四、实战案例索引

### 案例 1：着陆页整体设计
- 北极星：首次付费转化率（Free → Paid）
- 约束：复杂产品、用户注意力稀缺、无社会证明
- 张力：**丰富价值 vs 稀缺注意力**
- 结构与原则：①一句话钩子 ②展示产品输出 ③接近零试用成本

### 案例 2：Screen 3 报告展示模块
- 张力：**证明深度的丰富输出 vs 一眼可感知的价值**
- 结构与原则：①视觉编码替代文字 ②每份报告只露一个最强片段 ③真实数据渲染

### 案例 3：Prompt 撰写（来自 PROMPT_PHILOSOPHY.md）
- 张力：**具体指导 vs 通用适用**
- 结构与原则：①教思考方法而非给答案 ②示例强化原则而非提供模板
- 典型反模式：硬编码修复特定 case

### 案例 4：营销内容优化（Reddit 分析文章）
- 北极星：帖子引发 ASIN 互动（用户主动分享 ASIN 让你分析）
- 约束：Reddit 反广告文化、用户对 AI 工具的疲劳、内容必须独立有价值
- L0 张力：品类分析内容路径是否最优？→ 验证通过
- L1 张力：Reddit 先发 + iPhone 壳先发？→ 验证通过
- L2 张力（核心）：**证明力最强的 before/after 放在文章末尾 → 80% 读者看不到** → 解：移到开头 30 秒内
- L2 张力（次要）：ASIN 回复模板第一句暴露工具 → 解：改为"Took a closer look at your listing"
- L3 张力：工具化语言("buyer intent analysis scored 7/10") vs Reddit 自然语言 → 解：替换为口语化表达
- 启发：不跳层直接优化 L3 语言，应从 L0 验证方向开始

### 案例 5：Meta Prompt 项目重组（挖根本张力的典型过程）
- 北极星：让用户获得在任何情境下都能自主设计最优 Prompt 的能力
- 约束：5个文件60%重复、用户时间有限、Prompt 设计高度依赖情境
- 张力下钻过程：
  - 第一版张力："完整理论体系 vs 快速实用工具"
  - 统领性测试：即使完美平衡了理论和工具，"为什么5个文件都在试图解决同一个问题"仍然没解释 → **不够根本**
  - 追问为什么：为什么需要这么多版本？→ 因为每个版本都在试图抽象通用规律，但总觉得不够 → 为什么总觉得不够？→ 因为 Prompt 设计本质上依赖特定情境
  - 第二版张力："知识可传递性 vs 情境依赖性"
  - 统领性测试：消解这个张力后，文件如何组织、理论vs工具、去重vs独立、示例如何选——全部变成推论 → **这才是根本**
- 启发：第一版张力是 L1（策略选择），第二版才是 L0（方向性矛盾）
